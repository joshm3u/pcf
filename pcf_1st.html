Template description

Project Completion Form

The purpose of the Project Completion Form is ensure quality and track the handover of deployment activities to the Operational team.  

The Project Completion Form contains a checklist of activities that the operational tech will verify, such as NetLocal, fiber labeling, rack elevations, and device labeling. 

If need change is needed, please comment on the approval corresponce instead of rejecting the it. 

Approvers

**For Edge Sites:

L1 Approver - Edge Project Lead (peer review)
L2 Approver - Edge Projects Manager
L3 Approver - Edge DC Ops Tech
L4 Approver - Edge DC Ops Manager 

**For Cluster Sites:

L1 Approver - Edge Project Lead (peer review)
L2 Approver - Edge Projects Manager
L3 Approver - Cluster DCO or Cluster DCO Lead
L4 Approver - Cluster DCO Manager

*Aproval chain can be changed as needed

Checklist

List down all Project MCM’s  that were executed as part of this project, leading up to Project completion. i.e, Hardware install; Vendor Walkthrough, Cable installation; Cable Patching & Validation, etc 

Make sure to update the status of the check list before sending for approvals; send this for approval once all tasks in activity plan are completed. 

Checklist tasks requiring physical validation will be carried out during site audit vist if not completed during Project handoff and will be tracked via a physical validation TT if needed. 

Note: All related evidences/logs need to be updated in the Primary SIM worklog.
